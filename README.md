# Library-Management-System

# Домашнє завдання  

## Library Management System (In-Memory)

**Курс:** Backend розробка на Node.js  
**Дедлайн:** кінець 6-го тижня  
**Максимум:** 20 балів
**Бонус** 5 балів за реалізацію збереження у файловій системі (json файлики)

---

## 🎯 Мета

Розробити backend-застосунок для управління бібліотекою з використанням **Node.js, Express та TypeScript**, реалізувавши:

- REST API
- зберігання даних в оперативній памʼяті (in-memory)
- валідацію вхідних даних
- базову бізнес-логіку позики книг

---

## 📝 Опис завдання

Ви реалізуєте **Library Management System**, який дозволяє:

- створювати та керувати книгами
- створювати користувачів
- видавати книги в позику
- повертати книги
- переглядати список позик

❗ Дані зберігаються **в памʼяті застосунку** (`Map` або масиви).  
❗ Після перезапуску сервера всі дані втрачаються — це очікувана поведінка.

## 📦 Модель даних

| Поле | Тип | Опис |
| ---- | ---- | ---- |
| id | string | Унікальний ідентифікатор |
| title | string | Назва книги |
| author | string | Автор |

| isbn | string | Унікальний ISBN |
| available | boolean | Чи доступна книга |

---

### 👤 User

| Поле | Тип | Опис |
| ---- | ---- | ---- |
| id | string | Унікальний ідентифікатор |
| name | string | Імʼя користувача |
| email | string | Email |

---

### 🔄 Loan

| Поле | Тип | Опис |
| ---- | ---- | ---- |
| id | string | Унікальний ідентифікатор |

## 🌐 REST API

### 📘 Books

| Метод | Endpoint | Опис |
| ------ | --------- | ------ |
| GET | `/books` | Отримати список книг | DONE | WORKS
| GET | `/books/:id` | Отримати книгу за ID | DONE | WORKS 
| POST | `/books` | Створити книгу | DONE | WORKS
| PUT | `/books/:id` | Оновити книгу | DONE | WORKS
| DELETE | `/books/:id` | Видалити книгу | DONE |

## 📑 Валідація

- Усі `POST` та `PUT` запити **обовʼязково** валідовуються через **Zod**
- При помилці валідації API має повертати:
  - HTTP `400 Bad Request`
  - опис помилки

---

## 📌 Бізнес-логіка

### Видача книги

- Книгу **не можна видати**, якщо:
  - вона вже недоступна (`available = false`)
  - для неї існує активна позика
- При успішній видачі:
  - створюється `Loan` зі статусом `ACTIVE`
  - у книги `available = false`

---

### Повернення книги

- Статус позики змінюється на `RETURNED`
- Заповнюється `returnDate`
- Книга знову стає доступною (`available = true`)
